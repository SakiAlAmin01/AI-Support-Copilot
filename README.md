## 📌 Overview

**Smart Support Copilot** is an AI-powered internal support assistant designed to help Technical Support Engineers, QA Analysts, and DevOps teams analyze customer issues faster and more accurately.

The system accepts:

* 📄 Error Logs
* ⚙️ Configuration Files
* 📷 Application Screenshots
* 💬 Customer Complaints

and converts them into:

* Root Cause Analysis
* Structured Bug Reports
* Escalation Recommendations
* Sentiment Classification
* Developer-Ready Documentation

This significantly reduces troubleshooting time and improves support efficiency.

---

## ✨ Key Features

### 🔍 Intelligent Root Cause Detection

Automatically analyzes incoming tickets and identifies the most likely causes of failure.

**Output Includes:**

* Top probable causes
* Affected components
* Recommended fixes
* Debugging suggestions

---

### 📊 AI Sentiment Analysis

Prioritize support queues automatically.

| Sentiment     | Priority |
| ------------- | -------- |
| 🔥 Critical   | Highest  |
| 😡 Frustrated | High     |
| 😐 Neutral    | Normal   |
| 😊 Positive   | Low      |

---

### 🐞 Automated Bug Report Generation

Convert messy customer messages into clean developer tickets.

Generated fields:

* Title
* Description
* Steps to Reproduce
* Expected Result
* Actual Result
* Severity
* Environment

---

### ⚠️ Escalation Recommendation Engine

Automatically determines whether:

✅ Support Team can solve it

OR

🚨 DevOps / Engineering intervention is required

---

### 📑 Documentation Generator

Generate:

* Internal troubleshooting guides
* Client response templates
* Resolution notes
* Knowledge base articles

---

### 💾 Session Persistence

Store previous analyses locally and continue investigations without losing context.

## 🚀 Installation

### Clone Repository

```bash
git clone https://github.com/yourusername/smart-support-copilot.git

cd smart-support-copilot
```

### Install Dependencies

```bash
npm install
```

### Configure Environment Variables

Create a `.env` file:

```env
PORT=3000
API_KEY=YOUR_API_KEY
```

### Run Server

```bash
npm start
```

### Open Application

```bash
http://localhost:3000
```
